## ‚ö†Ô∏è Important Notes

1. **Administrator Privileges**: Port 443 requires administrator privileges on Windows
2. **Self-Signed Certificate**: The SSL certificate is self-signed, so browsers will show a warning
3. **Mixed Content**: The frontend (HTTPS) calls the backend (HTTP). Modern browsers may block this. Consider:
   - Enabling HTTPS on the backend as well, OR
   - Configuring your browser to allow mixed content for localhost

## üîß Troubleshooting

### Port 443 Already in Use
If you get an error that port 443 is already in use:
```powershell
# Find what's using port 443
netstat -ano | findstr :443

# Stop the process (replace PID with the actual process ID)
Stop-Process -Id <PID> -Force
```

### Browser Security Warning
This is expected with self-signed certificates. To proceed:
- Chrome/Edge: Click "Advanced" ‚Üí "Proceed to localhost (unsafe)"
- Firefox: Click "Advanced" ‚Üí "Accept the Risk and Continue"

### Mixed Content Warning
If the frontend can't call the backend API due to mixed content (HTTPS‚ÜíHTTP):
- Use browser developer tools to check console errors
- Consider running backend on HTTPS as well
- Or use a reverse proxy like nginx
